# 贡献指南

这个仓库的目标不是堆更多业务案例，而是持续把真实项目里跑出来的方法论提炼成可复用框架。

## 贡献原则

- 优先沉淀流程、规则、模板、错题本和协作方式。
- 新增内容尽量做“泛化”，不要只描述单个项目的私有上下文。
- 如果一条经验只能服务一个项目，优先放到示例或案例区，而不是核心框架区。
- 改规则时同时说明它在拦什么问题、替代什么旧做法。

## 提交流程

1. 明确这次改动属于流程、规则、角色、错题本还是样例文档。
2. 修改对应目录，并补足必要说明。
3. 如果引入了新的门禁或 checklist，同步更新 README 或相关索引。
4. 提交前检查是否仍然符合“可迁移、可接力、可复盘”的仓库定位。

## 目录约定

- `docs/workflows/`：主流程
- `docs/personas/`：角色定义
- `docs/rules/`：硬规则
- `docs/runbooks/error-books/`：错题本
- `docs/growth/`：愿景生长样例
- `docs/templates/`：模板

## 不建议提交的内容

- 与单一业务强绑定、无法复用的实现细节
- 没有经过抽象的聊天记录搬运
- 只描述“做了什么”但没有沉淀“为什么这样做”的文档
