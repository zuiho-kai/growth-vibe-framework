# 愿景生长样例

这一组文档直接来自 Greyfield 的“愿景生长”实践，用来展示一套项目如何在不做重规划的前提下，依然保持方向清晰、边界可控。

## 推荐阅读顺序

1. `architecture-v2.md`：长期愿景和系统位置
2. `context-runtime.md`：上下文运行时和核心思路
3. `spine-now.md`：当前冻结边界
4. `engineering-lessons.md`：工程教训与沉淀
5. `MAP.md` / `INDEX.md`：文档索引方式

## 这组文档的价值

- 它们不是模板语法，而是“如何让项目长出来”的真实样本。
- 它们说明了为什么“先冻结当前 spine”比“先写完整蓝图”更适合高不确定度项目。
- 它们可以直接作为你新项目的骨架参考，再按你自己的业务重新改写。
